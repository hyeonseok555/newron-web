## 5. API 클라이언트 레이어

### CORS 프록시 구조
- 브라우저에서 HTTP 백엔드 직접 접근 시 Mixed Content 에러 방지
- `app/api/proxy/[...path]/route.ts` 에서 경로 파싱 후 각 백엔드로 중계
  ```
  /api/proxy/main/news        →  https://newron.shop/api/v1/news
  /api/proxy/pers/api/v1/...  →  http://121.134.239.75:7000/api/v1/...
  /api/proxy/ai/...           →  http://121.134.239.75:8024/api/v1/...
  /api/proxy/brief/...        →  http://121.134.239.75:9000/...
  ```
- SSE(`text/event-stream`) 응답은 그대로 스트리밍 전달

### Axios 클라이언트 (`lib/api/clients.ts`)
- 서버사이드(SSR)에서는 `process.env.*`로 직접 접근
- 클라이언트사이드에서는 `/api/proxy/*` 경유
- 인증 인터셉터: `sessionStorage`의 `newron_access_token` 자동 주입

## 6. 인증 (NextAuth.js v5)

### 흐름
1. 사용자가 `/login`에서 "Google로 시작하기" 클릭
2. Google OAuth → `id_token` 수신
3. NextAuth `jwt` 콜백에서 `POST /auth/google`로 newron 서버 토큰 교환
4. `newron_access_token` + `user_id` → Session에 저장
5. 이후 API 요청에 Bearer 토큰 자동 첨부

### 게스트 모드
- `lib/guestId.ts`: 비로그인 사용자는 `crypto.randomUUID()`를 localStorage에 저장해 `user_id`로 사용
